    21.4. Swift의 Array는 값 타입인데, 이것이 성능과 메모리 사용에 어떤 영향을 미치나요?

	- List를 구현하는 방법으로 Array와 LinkedList가 있음. List는 데이터를 연속으로 저장하는 자료구조. Array로 구현시 물리적으로 인접해서 index 기반으로 접근이 가능하고, LinkedList로 구현시 논리적으로 인접해서 직접 접근이 불가하고 인접 node들을 거쳐서 접근해야함.
	1. 데이터 접근 속도는 Array가 빠르다.
	2. 중간에 원소 삽입과 삭제는 LinkedList가 빠르다. Array는 삽입, 삭제시 뒤에 있는 원소들을 옮겨주어야하는 반면 노드를 끊어서 새로운 Node에 연결을 해주면 되기 때문이다. 
	3. 🐮 Array는 메모리 효율적, Linked List는 포인터 때문에 추가 메모리 사용

		- 데이터가 붙어있어서 Array의 캐시 효율이 좋다.
		-  연속 메모리 (Array): 
			CPU가 array[0] 접근 시 → 주변 데이터(1,2,3,4,5) 캐시에 함께 로드 → array[1], array[2] 접근 시 캐시 히트 ✅
		- 분산 메모리 (LinkedList)
			CPU가 node1 접근 시: → node2는 다른 위치에 있음 → node2 접근 시 캐시 미스 ❌

### 동시성

32. 동시성(Concurrency)과 병렬성(Parallelism)의 차이점을 설명해주세요. 32.1. iOS 기기는 여러 개의 CPU 코어를 가지고 있는데, 이것이 동시성/병렬성 구현과 어떤 관련이 있나요? 32.2. 동시성 프로그래밍 시 발생할 수 있는 문제점은 무엇이며, 이를 해결하기 위한 방법에는 어떤 것들이 있나요?
	- 동시성은 하나의 스레드에서 여러 작업을 처리할 때 사용하는 용어이다. 작업이 끝나고 다음 작업을 이어가는 것을 동기적, 작업이 끝나는 것을 기다리지 않고 다음 작업을 진행하면서 이전 작업이 끝나길 기다리는 것을 비동기라고 한다. 보동 동시성은 "비동기 처리"에 대한 용어다. 병렬성은 여러개의 스레드에서 동시 작업하는 것을 말한다. 스레드가 하나인 경우 직렬, 스레드가 두개 이상인 경우 병렬이라고 말한다.
